원격 GitHub와 로컬 VS Code의 폴더명을 변경하려면 다음 순서로 진행하면 됩니다.

## 1\. 로컬 폴더 이름 변경

먼저 VS Code에서 작업 중인 로컬 폴더의 이름을 변경합니다. VS Code 외부(파일 탐색기/Finder)에서 폴더 이름을 직접 수정하세요. VS Code가 해당 폴더를 열고 있다면 닫고 진행하는 것이 좋습니다.

## 2\. GitHub 원격 저장소 이름 변경

GitHub 웹사이트에서 원격 저장소의 이름을 변경합니다.

1.  GitHub에 로그인하여 해당 저장소로 이동합니다.
2.  저장소 페이지에서 **Settings** 탭으로 들어갑니다.
3.  **General** 탭에서 **Repository name** 아래의 입력란에 새 이름을 입력하고 **Rename** 버튼을 클릭합니다.
4.  이 과정이 끝나면 GitHub가 새로운 원격 저장소 URL을 알려줍니다.

## 3\. 로컬 저장소 원격 URL 업데이트

이제 로컬 저장소가 변경된 원격 저장소의 위치를 알 수 있도록 URL을 업데이트해야 합니다. VS Code의 터미널을 열고 다음 명령어를 순서대로 실행하세요.

1.  **새 원격 URL 확인:**

    ```bash
    git remote -v
    ```

    이 명령어를 실행하여 현재 설정된 원격 저장소의 URL을 확인합니다.

2.  **기존 원격 URL 제거:**

    ```bash
    git remote remove origin
    ```

    기존의 `origin`이라는 원격 저장소 별칭을 제거합니다.

3.  **새 원격 URL 추가:**

    ```bash
    git remote add origin [새로운_원격_저장소_URL]
    ```

    GitHub에서 변경된 새로운 저장소 URL을 사용하여 `origin` 별칭을 다시 추가합니다.

이후 `git remote -v` 명령어로 새 URL이 올바르게 설정되었는지 다시 확인할 수 있습니다. 이제 로컬 저장소와 원격 저장소가 성공적으로 동기화됩니다.

## 4\. 변경 내용 푸시

마지막으로, 로컬에서 변경된 내용을 원격 저장소에 반영하기 위해 다음 명령어를 사용해 변경 사항을 푸시합니다.

```bash
git push -u origin main
```

만약 브랜치 이름이 `main`이 아니라면 `master` 등 실제 브랜치 이름으로 바꿔서 사용하면 됩니다.
